## การใช้งาน

```tsx
import { Fieldset, FieldsetLegend, Field, FieldLabel, FieldControl } from '@kairo-ui/react';

export function Example() {
  return (
    <Fieldset>
      <FieldsetLegend>Shipping address</FieldsetLegend>
      <Field>
        <FieldLabel>Full name</FieldLabel>
        <FieldControl />
      </Field>
      <Field>
        <FieldLabel>Email</FieldLabel>
        <FieldControl type="email" />
      </Field>
    </Fieldset>
  );
}
```

`Fieldset` render อีลีเมนต์ `<fieldset>` แบบ native จริง ๆ และ `FieldsetLegend` คือ
ป้ายกำกับของมัน — แต่ `FieldsetLegend` render `<div>` ธรรมดา ไม่ใช่ `<legend>` แบบ
native โดยเชื่อมกลับไปยัง fieldset ให้ assistive tech รับรู้ผ่าน `aria-labelledby`
แทนที่จะพึ่งความสัมพันธ์ fieldset/legend แบบ native ทำแบบนี้โดยตั้งใจ: `<legend>` จริง
มีปัญหาเรื่องสไตล์ที่ไม่แน่นอนข้ามเอนจินเบราว์เซอร์มานาน (หลายปีที่มันไม่สามารถเข้าร่วม
เลย์เอาต์ `display: flex`/`grid` ที่ตั้งไว้บน `<fieldset>` ผู้ปกครองได้อย่างน่าเชื่อถือ
และ UA stylesheet ของทุกเอนจินก็ยังคร่อมเส้นขอบของ fieldset อยู่ดี) การให้มันเป็นอีลีเมนต์
ธรรมดาที่เชื่อมด้วย ARIA จึงทำให้ styling ยังคงง่ายอยู่

เพราะ `Fieldset` render `<fieldset>` จริง การตั้งค่า prop `disabled` จึงปิดใช้งาน
ทุกตัวควบคุมที่โฟกัสได้ซึ่งซ้อนอยู่ข้างในแบบ **native** — ไม่ว่าจะเป็นอินพุตข้อความ ปุ่ม
select หรือคอมโพเนนต์ของ Kairo ตัวไหนก็ตามที่สร้างบน Base UI root — โดยไม่ต้องต่อสายเพิ่ม
ต่อตัวควบคุมเลย แม้แต่ตัวควบคุมที่ Kairo เองไม่รู้จักเป็นพิเศษก็ตาม

`Fieldset` เป็นตัวคอนเทนเนอร์ตามธรรมชาติสำหรับ `Field` หนึ่งตัวหรือมากกว่า — แต่ละตัวผูก
ป้ายกำกับ ตัวควบคุม และคำอธิบาย/ข้อความแจ้งข้อผิดพลาดที่เป็นทางเลือกเข้าด้วยกัน — แต่มันก็
จัดกลุ่มอีลีเมนต์อะไรก็ได้ที่อยู่ข้างในเช่นกัน ดังนั้น markup ฟอร์มธรรมดาก็ใช้ได้เหมือนกัน

## Props

### `Fieldset`

| Prop        | Type              | Default | Description                                                       |
| ----------- | ----------------- | ------- | ---------------------------------------------------------------------- |
| `disabled`  | `boolean`         | `false` | ปิดใช้งานทุกตัวควบคุมที่โฟกัสได้ซึ่งซ้อนอยู่ในฟีลด์เซ็ตแบบ native            |
| `className` | `string`          | —       | รวมกับคลาสพื้นฐาน `kairo-fieldset`                                       |
| ...         | Additional props | —       | props อื่น ๆ ที่เหลือจะถูกส่งต่อไปยังอีลีเมนต์ `<fieldset>` ด้านใน           |

### `FieldsetLegend`

| Prop        | Type              | Default | Description                                                  |
| ----------- | ----------------- | ------- | -------------------------------------------------------------------- |
| `className` | `string`          | —       | รวมกับคลาสพื้นฐาน `kairo-fieldset-legend`                              |
| ...         | Additional props | —       | props อื่น ๆ ที่เหลือจะถูกส่งต่อไปยังอีลีเมนต์ด้านใน                    |

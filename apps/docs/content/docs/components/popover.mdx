## Usage

```tsx
import { Popover, PopoverTrigger, PopoverContent, PopoverTitle, PopoverDescription, PopoverClose } from '@kairo-ui/react';

export function Example() {
  return (
    <Popover>
      <PopoverTrigger>Share</PopoverTrigger>
      <PopoverContent>
        <PopoverTitle>Share this document</PopoverTitle>
        <PopoverDescription>Anyone with the link will be able to view this.</PopoverDescription>
        <PopoverClose>Done</PopoverClose>
      </PopoverContent>
    </Popover>
  );
}
```

`Popover` groups the parts (it renders no DOM element of its own — the same as
Base UI's `Popover.Root`). `PopoverContent` composes Base UI's
`Popover.Portal` > `Popover.Positioner` + `Popover.Popup` in one call, so you
don't need to reach for those parts yourself. Anchored positioning,
Escape/outside-press dismissal, and `role="dialog"` all come from
`@base-ui/react`.

## Props

### `Popover`

| Prop           | Type                                     | Default | Description                                                                                    |
| -------------- | ------------------------------------------ | ------- | ------------------------------------------------------------------------------------------------ |
| `open`         | `boolean`                                  | —       | Controlled open state. Use with `onOpenChange`.                                                  |
| `defaultOpen`  | `boolean`                                  | `false` | Initial open state for an uncontrolled popover.                                                  |
| `modal`        | `boolean \| 'trap-focus'`                  | `false` | Whether the popover traps focus/locks scroll (`true`), only traps focus (`'trap-focus'`), or neither (`false`). |
| `onOpenChange` | `(open: boolean, eventDetails) => void`    | —       | Called when the popover opens or closes.                                                         |
| ...            | Additional props | — | All other props are forwarded through.                                       |

### `PopoverTrigger` / `PopoverClose`

Render a native `<button>` by default, or compose with another element via
`render` (e.g. `<PopoverTrigger render={<Button>Open</Button>} />`).

### `PopoverContent`

| Prop         | Type                                                          | Default    | Description                                                                    |
| ------------ | -------------------------------------------------------------- | ---------- | ---------------------------------------------------------------------------------- |
| `side`       | `'top' \| 'bottom' \| 'left' \| 'right' \| 'inline-end' \| 'inline-start'` | `'bottom'` | Which side of the trigger to render the popup against.                             |
| `align`      | `'start' \| 'center' \| 'end'`                                  | `'center'` | How to align the popup relative to `side`.                                          |
| `sideOffset` | `number`                                                        | `8`        | Distance in pixels between the trigger and the popup.                              |
| `className`  | `string`                                                        | —          | Merged with the base `kairo-popover-popup` class on the popup element.              |
| `initialFocus` | `boolean \| RefObject \| function`                            | —          | Element to focus when the popover opens. Defaults to the first tabbable element inside the popup, or the popup itself when opened via touch (to avoid triggering the on-screen keyboard). Accepts `false` (leave focus untouched), `true` (use the default), a `RefObject` (focus that element), or a function — called with the interaction type (`'mouse'`, `'touch'`, `'pen'`, `'keyboard'`) — that returns an element, `true`/`null` for the default, or `false`/`undefined` to do nothing. |
| `finalFocus` | `boolean \| RefObject \| function`                              | —          | Element to focus when the popover closes. Defaults to the trigger, or whichever element was focused before it opened. Accepts `false` (leave focus untouched), `true` (use the default), a `RefObject` (focus that element), or a function — called with the interaction type (`'mouse'`, `'touch'`, `'pen'`, `'keyboard'`) — that returns an element, `true`/`null` for the default, or `false`/`undefined` to do nothing. |
| ...          | Additional props | — | All other props are forwarded to the underlying popup element. |

### `PopoverTitle` / `PopoverDescription`

Render an `<h2>`/`<p>` respectively, linked to the popup via
`aria-labelledby`/`aria-describedby` automatically.

### `PopoverArrow`

Optional pointer rendered between the popup and its trigger. Place it among
`PopoverContent`'s children; Base UI positions it automatically, and
`@kairo-ui/theme` draws the pointer shape off the `data-side` attribute Base
UI sets on it.

## Usage

```tsx
import { Tooltip } from '@kairo-ui/react';

export function Example() {
  return (
    <Tooltip content="Saved to your library">
      <button type="button">Hover me</button>
    </Tooltip>
  );
}
```

`children` is the trigger element — Kairo renders it directly (via Base UI's
`render` composition) and merges the hover/focus interaction handlers onto it,
so no extra DOM wrapper is introduced. `content` is the popup's contents.

If you render many tooltips, wrap a subtree once in `<TooltipProvider>` so
adjacent tooltips share open/close delay timing (hovering from one trigger to
the next then opens instantly); a single `<Tooltip>` works fine without one.

## Props

### `Tooltip`

| Prop         | Type                                                          | Default    | Description                                                                    |
| ------------ | -------------------------------------------------------------- | ---------- | ---------------------------------------------------------------------------------- |
| `children`   | `ReactElement`                                                  | —          | The trigger element. Required.                                                     |
| `content`    | `ReactNode`                                                     | —          | The popup's contents. Required.                                                    |
| `side`       | `'top' \| 'bottom' \| 'left' \| 'right' \| 'inline-end' \| 'inline-start'` | `'top'`    | Which side of the trigger to render the popup against.                             |
| `align`      | `'start' \| 'center' \| 'end'`                                  | `'center'` | How to align the popup relative to `side`.                                          |
| `sideOffset` | `number`                                                        | `8`        | Distance in pixels between the trigger and the popup.                              |
| `delay`      | `number`                                                        | `600`      | How long to wait before opening the tooltip, in ms.                                |
| `closeDelay` | `number`                                                        | `0`        | How long to wait before closing the tooltip, in ms.                                 |
| `open`       | `boolean`                                                       | —          | Controlled open state. Use with `onOpenChange`.                                     |
| `defaultOpen`| `boolean`                                                       | `false`    | Initial open state for an uncontrolled tooltip.                                     |
| `onOpenChange` | `(open: boolean, eventDetails) => void`                       | —          | Called when the tooltip opens or closes.                                            |
| `className`  | `string`                                                        | —          | Merged with the base `kairo-tooltip-popup` class on the popup element.              |
| ...          | Additional props | —          | All other props (e.g. `trackCursorAxis`, `disabled`) are forwarded through. |

### `TooltipProvider`

Groups multiple `Tooltip`s so they share open/close delay timing: once one
tooltip in the group has opened, an adjacent one opens instantly (skipping its
own `delay`) if it's triggered within `timeout` ms of the previous one
closing.

| Prop         | Type              | Default | Description                                                                                                                                   |
| ------------ | ----------------- | ------- | ---------------------------------------------------------------------------------------------------------------------------------------------------- |
| `delay`      | `number`          | —       | How long to wait before opening a tooltip in the group, in ms. Falls back to each `Tooltip`'s own `delay` (`600` by default) when unset.               |
| `closeDelay` | `number`          | —       | How long to wait before closing a tooltip in the group, in ms. Falls back to each `Tooltip`'s own `closeDelay` (`0` by default) when unset.            |
| `timeout`    | `number`          | `400`   | If another tooltip in the group opens within this many ms of the previous one closing, it opens instantly instead of waiting out `delay`.              |
| `children`   | `React.ReactNode` | —       | The tooltips (and any other content) in the group.                                                                                                      |

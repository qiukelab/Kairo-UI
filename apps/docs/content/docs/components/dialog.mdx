## Usage

```tsx
import { Dialog, DialogTrigger, DialogContent, DialogTitle, DialogDescription, DialogClose } from '@kairo-ui/react';

export function Example() {
  return (
    <Dialog>
      <DialogTrigger>Edit profile</DialogTrigger>
      <DialogContent>
        <DialogTitle>Edit profile</DialogTitle>
        <DialogDescription>Make changes to your profile here.</DialogDescription>
        <DialogClose>Close</DialogClose>
      </DialogContent>
    </Dialog>
  );
}
```

`Dialog` groups the parts (it renders no DOM element of its own — the same as
Base UI's `Dialog.Root`). `DialogContent` composes Base UI's
`Dialog.Portal` > `Dialog.Backdrop` + `Dialog.Popup` in one call, so you don't
need to reach for those parts yourself. Focus trapping, scroll locking,
Escape/outside-press dismissal and `role="dialog"` all come from
`@base-ui/react`.

## Props

### `Dialog`

| Prop           | Type                                                  | Default | Description                                                                                            |
| -------------- | ------------------------------------------------------ | ------- | ----------------------------------------------------------------------------------------------------------- |
| `open`         | `boolean`                                               | —       | Controlled open state. Use with `onOpenChange`.                                                              |
| `defaultOpen`  | `boolean`                                               | `false` | Initial open state for an uncontrolled dialog.                                                               |
| `modal`        | `boolean \| 'trap-focus'`                               | `true`  | Whether the dialog traps focus/locks scroll (`true`), only traps focus (`'trap-focus'`), or neither (`false`). |
| `onOpenChange` | `(open: boolean, eventDetails) => void`                 | —       | Called when the dialog opens or closes.                                                                      |
| ...            | Additional props | — | All other props are forwarded through.                                                    |

### `DialogTrigger` / `DialogClose`

Render a native `<button>` by default, or compose with another element via
`render` (e.g. `<DialogTrigger render={<Button>Open</Button>} />`).

### `DialogContent`

| Prop          | Type                                              | Default | Description                                                              |
| ------------- | ---------------------------------------------------- | ------- | ------------------------------------------------------------------------------ |
| `className`   | `string`                                             | —       | Merged with the base `kairo-dialog-popup` class on the popup element.          |
| `initialFocus`| `boolean \| RefObject \| function`                   | —       | Element to focus when the dialog opens. Defaults to the first tabbable element inside the popup, or the popup itself when opened via touch (to avoid triggering the on-screen keyboard). Accepts `false` (leave focus untouched), `true` (use the default), a `RefObject` (focus that element), or a function — called with the interaction type (`'mouse'`, `'touch'`, `'pen'`, `'keyboard'`) — that returns an element, `true`/`null` for the default, or `false`/`undefined` to do nothing. |
| `finalFocus`  | `boolean \| RefObject \| function`                   | —       | Element to focus when the dialog closes. Defaults to the trigger, or whichever element was focused before it opened. Accepts `false` (leave focus untouched), `true` (use the default), a `RefObject` (focus that element), or a function — called with the interaction type (`'mouse'`, `'touch'`, `'pen'`, `'keyboard'`) — that returns an element, `true`/`null` for the default, or `false`/`undefined` to do nothing. |
| ...           | Additional props | — | All other props are forwarded to the underlying popup element. |

### `DialogTitle` / `DialogDescription`

Render an `<h2>`/`<p>` respectively, linked to the popup via
`aria-labelledby`/`aria-describedby` automatically.
